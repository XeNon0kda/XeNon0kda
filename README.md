###  Мои проекты

Вот несколько проектов, которые я считаю показательными. В каждом есть graceful shutdown, Docker-контейнеризация, чистый код и чистая архитектура.

#### 1. Сервис сокращения ссылок (URL Shortener)
*Кратко:* Сервис, который превращает длинные URL в короткие.
*   **Стек:** Go, Docker.
*   **Фишки:** Чистая архитектура (слои: handler, service, storage). Graceful shutdown обрабатывает сигналы системы. Код легко расширять.

#### 2. Мониторинг доступности сайтов (Site Monitor)
*Кратко:* Утилита для проверки статуса (HTTP 200/500) и времени ответа веб-сайтов.
*   **Стек:** Go, Docker.
*   **Фишки:** Конкурентность (горутины для параллельной проверки сайтов). Логирование. Конфигурация через переменные окружения или файл.

#### 3. Мониторинг системных метрик (System Metrics Collector (sysmc))
*Кратко:* Агент для сбора данных о состоянии компьютера (CPU, RAM, диск).
*   **Стек:** Go, Docker.
*   **Фишки:** Взаимодействие с системными вызовами ОС. Периодический сбор данных (тикеры). Архитектура позволяет легко добавить отправку метрик на удаленный сервер.

#### 4. Дипломный проект: Мессенджер (Work in Progress) 
*Кратко:* Real-time мессенджер, работающий через WebSocket.
*   **Статус:** Сейчас нахожусь в процессе активного рефакторинга. Проект начинался как прототип, чтобы быстро проверить концепцию и работоспособность WebSocket-соединений. Сейчас я привожу его к той же архитектурной чистоте, что и остальные проекты.
*   **Технологии внутри:** WebSocket, обработка конкурентных подключений, комнаты (чаты), Graceful shutdown.
